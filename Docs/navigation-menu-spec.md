# カタヤマ WEBサイト

情報設計／メニュー構成・リンク仕様書（ドラフト）

## 1. サイト全体の方針

* ビジネスゴール

  * **大規模修繕の問い合わせ増**（管理組合・オーナー）
  * **リフォーム相談の問い合わせ増**（個人住戸・店舗）
* 情報設計の方針

  * グローバルナビは「サービス／実績／会社情報／問い合わせ」の**王道構成**でシンプルに。
  * ペルソナ別（大規模修繕を検討／リフォームを検討）は、**トップページの入口＆サービス詳細ページ内で案内**する「ガイド役」にする。
  * 大規模修繕ページには、現行 `/repair` の内容をベースに、**ページ内ローカルメニュー（アンカーリンク※）＋関連メニュー**を設置する。([katayama-square.co.jp](https://katayama-square.co.jp/repair/))

> ※アンカーリンク：ページ内の特定セクションにジャンプするリンク

---

## 2. ペルソナ

1. **大規模修繕を検討している方**

   * 管理組合理事長・理事・マンションオーナー・管理会社担当者など
   * ニーズ：長期修繕計画、工事の流れ、合意形成、安全／防犯、費用感、実績

2. **リフォームを検討している方**

   * 分譲マンション区分所有者、戸建てオーナー、店舗オーナー
   * ニーズ：Before/After、費用・工期、生活への影響、デザイン性、実績

## 4. ペルソナ別入口（トップページ）

### 4-1. 配置場所

* トップページのヒーロー（メインビジュアル）の直下に配置。
* 視線の流れとして、「映像 → キャッチコピー → ペルソナ選択」の順。

### 4-2. コンポーネント仕様

| 要素   | 文言案                    | リンク先                               | 意図                                             |
| ---- | ---------------------- | ---------------------------------- | ---------------------------------------------- |
| 見出し  | 「あなたのご状況に近いものをお選びください」 | –                                  | 自分ごと化を促し、クリックのきっかけを作る。                         |
| カードA | 「大規模修繕を検討している方へ」       | `/services/repair#for-management`  | 管理組合・オーナー向けのストーリーLPへ誘導。大規模修繕ページ内の専用セクションにジャンプ。 |
| カードB | 「リフォームを検討している方へ」       | `/services/reform#for-reform-user` | 個人・店舗リフォーム向けLPへ誘導。                             |

> ※`#for-management` や `#for-reform-user` はアンカーIDの例。実装時に変更可。

## 5. 大規模修繕ページ（/services/repair）

### 5-1. ページ全体の役割

* ペルソナ：大規模修繕を検討している管理組合・オーナー・管理会社。
* 役割：

  * 大規模修繕とは何か／いつやるのかを理解してもらう。
  * カタヤマに任せるメリット（技術・安全・防犯）を伝える。([katayama-square.co.jp](https://katayama-square.co.jp/repair/))
  * 工事の流れをイメージさせ、不安を減らす。
  * 実績を見せて信頼を上げ、問い合わせへつなぐ。

## 7. 共通リンク仕様（概要）

* ロゴ：全ページ共通で `/`（ホーム）へ。
* フッターナビ（例）：

  * ホーム / サービス / 大規模修繕 / リフォーム / 施工実績 / 会社情報 / お客様の声 / 採用情報 / 協力業者募集 / プライバシーポリシー / お問い合わせ
  * 既存フッターの項目構成を踏襲しつつ整理。([katayama-square.co.jp](https://katayama-square.co.jp/repair/))
* 外部リンク：

  * Instagram、新卒採用サイト、メンバーサイト等は、外部アイコン＋新しいタブで開く仕様を明示。

## この仕様書の使い方イメージ

* **ディレクター／デザイナー向け**
  → サイトマップ・ワイヤーフレーム作成のベースに。
* **エンジニア向け**
  → ルーティング・アンカーID・グローバル／ローカルナビ実装の仕様に。
* **Claude Code向け**
  → 「各セクションのID・ラベル」をこのままプロンプトに貼り付けて、コンポーネント分割指示に使う。
